## 🔧 防火墙配置

### 需要开放的端口

```bash
# 开放前端端口（用户访问）
sudo firewall-cmd --permanent --add-port=3000/tcp

# 开放后端端口（API 访问）
sudo firewall-cmd --permanent --add-port=8000/tcp

# 重载防火墙配置
sudo firewall-cmd --reload

# 查看已开放端口
sudo firewall-cmd --list-ports
```

### 验证端口开放

```bash
# 从其他机器测试端口连通性
telnet 10.119.133.236 3000
telnet 10.119.133.236 8000

# 或使用 curl
curl http://10.119.133.236:3000
```

## ⚠️ 常见问题

### Q1: 无法访问前端界面？

**检查清单**:
1. ✅ 服务是否启动？
   ```bash
   ./scripts/status.sh
   ```

2. ✅ 防火墙是否开放端口？
   ```bash
   sudo firewall-cmd --list-ports
   ```

3. ✅ 网络是否连通？
   ```bash
   ping 10.119.133.236
   ```

### Q2: 提示"连接被拒绝"？

**可能原因**:
- 服务未启动
- 防火墙阻止

**解决方法**:
```bash
# 1. 启动服务
./scripts/start.sh

# 2. 开放端口
sudo firewall-cmd --permanent --add-port=3000/tcp
sudo firewall-cmd --permanent --add-port=8000/tcp
sudo firewall-cmd --reload
```

### Q3: 页面加载很慢？

**可能原因**:
- 网络延迟高
- 服务器资源不足

**优化建议**:
- 使用相同局域网访问
- 检查服务器 CPU/内存使用率
